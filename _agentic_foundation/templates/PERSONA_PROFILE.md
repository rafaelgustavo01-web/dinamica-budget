# PERSONA_PROFILE.md

## Active Personas
Select 1 core persona and up to 2 supporting personas from `_agentic_foundation/personas/`.

Recommended pattern:
- one delivery persona
- one control persona
- one optimization persona

Examples:
- backend + security + performance
- frontend + ux + qa
- architect + devops + security
- sap + architect + qa

## Rule
Persona changes behavior and review lens.
Stack changes implementation constraints.
