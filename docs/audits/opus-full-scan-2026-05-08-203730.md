## 4. Achados Detalhados — P0 (Bloqueador)

### #1 — Contrato `origem` divergente (P0)

**Backend** (`app/backend/schemas/servico.py:18`):
```python
origem: str | None = None  # None for BaseTcpo items
```

**Frontend** (`app/frontend/src/shared/types/contracts/servicos.ts:13`):
```ts
origem: 'TCPO' | 'PROPRIA';   // não-nullable, enum literal
```

**Impacto técnico:**
- TS aceita o response em compile-time porque o tipo é `unknown` ao chegar do `fetch`. Em runtime, qualquer iteração ou render que faça `servico.origem.toLowerCase()` ou `servico.origem === 'TCPO'` em listagem mista (TCPO + ItemProprio) recebe `null` e quebra.
- A `ServicesPage` foi reescrita justamente para listar serviços TCPO + próprios; é exatamente o cenário em que o bug aflora.

**Correção (preferida):** ajustar back para retornar enum string sempre populado (ex.: `origem='TCPO'` para itens BaseTcpo) ou ajustar front para `origem: 'TCPO' | 'PROPRIA' | null`. Se houver tempo até a demo, a correção front é menor (1 linha) e segura.

## 5. Achados Detalhados — P1 (Hotfix curto prazo)

### #2 — Migration 026: sequência global e SET DEFAULT
- Sequência `operacional.item_proprio_seq` é **global**, não por cliente. Se a regra de negócio é "código único por cliente", `nextval` global serve, mas `BaseTcpo.codigo_origem UNIQUE` é global; `ItemProprio` não documenta a constraint composta.
- `ALTER TABLE … SET DEFAULT` aplica apenas a novos inserts. OK desde que registros existentes já tenham `codigo_origem` preenchido (caso contrário, NOT NULL viola).
- **Ação:** confirmar que `itens_proprios.codigo_origem` já tem dados não-nulos antes da migração rodar em prod; idealmente adicionar `UNIQUE(cliente_id, codigo_origem)` se ainda não existir.

### #3 — `scripts/run_alembic.py`
```python
sys.path.insert(0, r'C:\DinamicaBudget\venv\Lib\site-packages')
os.chdir(r'C:\DinamicaBudget')
```
Inutilizável fora da máquina/servidor IIS específico. **Ação:** mover para `scripts/win/` com nome explícito (ex.: `run_alembic_iis.py`) e adicionar comentário; ou parametrizar via env var.

### #4 — `servico_catalog_service._explode_recursivo_tcpo`
`visited=set()` reinstanciado nas chamadas recursivas → ciclos A→B→A passam. **Ação:** propagar o mesmo `visited` na recursão e adicionar `max_depth` defensivo.

### #5/#6 — `endpoints/admin.py` background embeddings
- `except Exception` engole tudo silenciosamente; falha de embedding não emite alerta acionável.
- Flag `recomputar_embeddings` do request é sobrescrita para `False` na chamada de ETL — comportamento divergente do que o cliente espera.
- **Ação:** logar `error=` com `exc_info=True`, propagar request flag, registrar métrica de falha.

### #7/#8 — `ServicesPage.tsx`
- Linha 6: remover `Box` da import.
- Linhas 108-111: substituir `parseFloat(String(item.custo_total)) || 0` por checagem explícita de null/undefined; `String(undefined)` → `"undefined"` → `NaN` → `0` mascara dados ruins.

### #9 — `app/frontend/dist_new/` versionado
- `.gitignore` cobre `dist/` e `frontend/dist/`, **não cobre `dist_new/`**. Confirmado pelo `web.config` que `dist_new/` é o root servido pelo IIS.
- Decidir: manter versionado (deploy artefato) ou mover para pipeline. Se manter:
  - documentar no README que `dist_new/` é o artefato deploy oficial;
  - garantir que `npm run build` antes do commit (caso contrário UI regride);
  - excluir `node_modules`, `*.map` se estiverem caindo no bundle.

## 6. Riscos Específicos das Mudanças Recentes

1. **Demo pode quebrar em `ServicesPage`** quando o backend devolve qualquer serviço TCPO sem `origem` (achado #1). Mitigação rápida: forçar `origem='TCPO'` no `ServicoTcpoResponse` para BaseTcpo no `servico_catalog_service`.
2. **Inconsistência src↔dist** em `dist_new/`. Não há garantia que o bundle reflita a última versão de `src/`. Antes da demo, validar com `git status` e `npm run build` em ambiente reprodutível, comparando hashes dos chunks (`servicesApi-*.js`, `searchApi-*.js`).
3. **Migration 026 em prod** sem revisar dados existentes pode quebrar se houver `itens_proprios.codigo_origem` nulo. Mesmo que `SET DEFAULT` apenas afete novos inserts, é prudente rodar em staging primeiro.
4. **Background embeddings silencioso** — após import via `/admin/etl`, busca semântica fica degradada por horas sem ninguém perceber.
5. **`run_alembic.py` em CI** falhará se algum job tentar usá-lo.
